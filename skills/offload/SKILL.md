# offload — выгрузка по ходу

Одна команда, секунды. НЕ отчёт, НЕ список, НЕ пересказ.

```bash
tide offload <session-slug> --cursor "<где стою — одна строка>" "<что решил / что дальше — одна строка>"
```

## Правила (жёсткие)

1. **По одной строке на каждое.** Курсор = «где стою». Заметка = «что решил / что
   дальше». Потянуло на абзац — режь: подробности уже лежат в workspace.
2. **Своя сессия.** `<session-slug>` — арка, в которой ведёшь работу (её назвал сид;
   сомневаешься — `tide status`, твоя — активная в текущем треде).
3. **Момент:** принял решение · свернул на развилке · закончил кусок · пинок хука.
   Пинок пришёл — выполни его команду сразу и заканчивай ход, не спорь.
4. Это дополнение к работе в арке, не замена: файлы работы — в workspace по ходу
   (правило [[tide-flow]]), offload — только курсор+строка контекста.

## Зачем

Качество выгрузки падает с длиной сессии — в конце выгружаешь уставшим и сжатым.
По-ходовые строки делают финальный хендофф курацией готового: сид собирается из
уже лежащей правды (карта входа), ничего не вспоминается в последний момент.
